# 자료구조 나만의 정의
자료구조라고 해서 거창한게 아니다. 
1 단계 :  상황에 필요한 기능을 정의 한다.
    - 예시 : 유통기한 관리를 위해 먼저 들어온게 먼저 나가야해

2 단계 :  핵심 가치를 판 단한다.
    - 예시 : 난 어차피 맨 앞에서만 물건을 꺼낼거고, 정리해둔 물품 중간에 무언갈 추가할 일이 많아  

3 단계 :  설게도를 짠다.
    - 자료구조의 각 개념을 안다면 여기서 내가 알아서 조합 할 수 있다.
    (힙, 트리, 해시 등)

3.5 단계 : 고오급 설계도 짜기
    - 지금 에시는 필요없지만 조건이 까다로운 경우, 특정 알고리즘이 들어가는등 좀 더 구체적인 설계도이다. (레드블랙트리 등)

4단계 : 물리적 구현
    - 예시 : 어? 그러면 배열 기반이 아닌 연결 기반이어야 하겠네?  
    (자바로 치면 구현된 클래스 = 링크드리스트) 

이렇게 끝이다. 여기서 가장 근본인 연결이냐 배열이냐의 차이만 명확하게 알면 된다.  
다만, 가치 판단을 할 때 여러 조건들이 붙을 수 있다.   
예로들어 중복이 없음 + 순서 필요없음 + 검색속도가 중요함 = hashSet(4단계) 되는거임  
해시 개념알지? 그럼 간단하지 바로 기본은 배열, 해시충돌 해결은 연결 또는 트리구로 가는거지

## 자료구조 핵심 5개
1. 추상 자료형 (Abstract Data Type, ADT)
2. 자료구조 (DataStructure)
3. 구현체 (Implementation)
3. 배열기반 (Array)
4. 연결기반 (Linked)

## 왜 핵심?
1. 추상 자료형과 자료구조를 차이를 알아야 자료구조 생태계를 이해하게 된다.  
이해를 못하면 암기로 가야하며 암기를 하면 응용이 어렵다.

2. 모든 자료구조에 무조건 들어가는 핵심이 딱 저 두개(배열, 연결)로 만들어진다.  
배열기반과 연결기반의 차이만 정확하게 알아도 앞으로 배우는 자료구조를 금방 파악 할 수 있다.

## ADT 란? 1단계임
우리가 구현할 자료구조의 추상적인 개념이다. 즉, 구현할 자료구조의 기능 명세서다.  
예로들면 처음 들어간게 가장 먼저 나와야함. (우린 이걸 스택이라고 한다.)

## 자료구조란? 2 ~ 3.5 단계임
데이터를 효율적으로 저장, 수정, 출력(접근 or 조회)하기 위해 어떻게 구현할지 정의한 방법론이다.
즉, 어떻게 구현 할지 구체적인 방식/설계를 하는것.  
예로들면 가장 기본적인 배열기반이냐, 연결기반이냐를 선택하거나  
조건이 까다로운 경우 응용적인 자료구조인 트리, 해시, 힙이 정해잠
(예들도 결국 배열 또는 연결기반 또는 둘다 써서 만들어짐)
