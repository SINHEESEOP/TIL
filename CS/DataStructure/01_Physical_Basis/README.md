# 01. 물리적 기반 (Physical Basis)

> **"모든 자료구조는 결국 배열(Array) 또는 연결(Linked)로 만들어진다."**

## 핵심 개념

자료구조를 아무리 복잡하게 설계해도, 메모리에 실제로 저장하는 방식은 딱 **두 가지**뿐이다.

| 방식 | 특징 | 예시 |
| :--- | :--- | :--- |
| **배열 (Array)** | 연속된 메모리 공간, 인덱스로 O(1) 접근 | `ArrayList`, 힙(Heap) |
| **연결 (Linked)** | 흩어진 메모리, 포인터로 연결 | `LinkedList`, 트리 노드 |

## 작성 예정

- `Array_Theory.md` - 배열의 메모리 구조와 특성
- `LinkedList_Theory.md` - 연결 리스트의 메모리 구조

## 왜 중요한가?

[자료구조 개념서](../00_Core_Concept/README.md)의 4단계 정의에서 설명했듯이, **가치 판단(2단계)** 에 따라 **물리적 구현 방식(4단계)** 이 결정된다.

예를 들어:
- "중간 삽입/삭제가 빈번하다" → **연결 기반** 선택
- "랜덤 접근이 중요하다" → **배열 기반** 선택
